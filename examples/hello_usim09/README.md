# hello_usim09 — Micro-C 6809 Hello World for usim09

The simplest possible mc09 program: prints a greeting and a count to the
MC6850 ACIA, runs under the usim09 6809 simulator.

## Quick start

```sh
make        # compile hello.c -> hello.HEX
make run    # run under usim09
```

Expected output:

```
Hello from Micro-C 6809!
Count: 1 2 3 4 5
```

## What the build does

```
cc09 hello.c -Iq
```

`cc09` is the mc09 command coordinator. It runs the full pipeline in one
step using the three environment variables set by the Makefile:

| Variable | Value | Purpose |
|----------|-------|---------|
| `MCDIR` | `../..` | Toolchain root — cc09 finds mcc09, asm09, slink here |
| `MCINCLUDE` | `../../include` | Header search path — finds `<6809io.h>` etc. |
| `MCLIBDIR` | `../../targets/usim09/lib09` | Library and startup files for this target |

The `-I` flag requests Intel HEX output (`hello.HEX`). The `-q` flag
suppresses the intermediate file listing.

## What the target provides

The `usim09` target (`targets/usim09/`) was generated by `mktarget` from
`targets/usim09/usim09.cfg`. The generated `lib09/` directory contains:

| File | Purpose |
|------|---------|
| `6809RLP.ASM` | Startup prefix: `ORG $E000`, `LDS #$7F00`, `JSR main`, `BRA *` spin |
| `6809RLM.ASM` | Global variable region marker: `ORG $0100` |
| `6809RLS.ASM` | Startup suffix: heap marker, vector table at `$FFF0` (RESET -> `$E000`) |
| `SERIO.ASM` | MC6850 ACIA driver: `putstr`, `putchr`, `putch`, `getchr` |
| `*.ASM` | Runtime library: multiply, divide, shift, comparisons |
| `EXTINDEX.LIB` | Symbol index for `slink` |

## Memory map

```
$0000-$00FF  Zero page (reserved, not used by mc09)
$0100-$7EFF  User RAM — globals start at $0100, heap grows up
$7F00        Stack top (stack grows downward)
$8000-$BFFF  Unassigned in usim09
$C000-$C001  MC6850 ACIA ($C000=status, $C001=data)
$E000-$FFF0  ROM — startup code + user program + runtime library
$FFF0-$FFFF  6809 hardware vector table (all vectors -> $E000)
```

## How usim09 handles I/O

usim09 emulates the MC6850 by mapping it to host stdin/stdout. `putchr`
polls `$C000` for TDRE (bit 1), then writes to `$C001`. From the program's
perspective it's a real UART; from the host's perspective it's just
characters going to stdout.

## How the program exits

After `main()` returns the startup spins (`BRA *`). usim09 does not have a
way of progmatically exiting. You can exit from the terminal by typing ~.
however the spin tends to io lock the simulator terminal.  'make run' causes
the program to end 1 second after it runs and the exicution ends.


```sh
echo "" | usim09 hello.HEX
```

To exit interactively if you run usim09 directly without the pipe: **Ctrl+C**.

SWI cannot be used as an exit mechanism because all hardware vectors —
including the SWI vector — point to `?begin` (`$E000`). Executing SWI
would restart the program rather than terminating the simulator.

## Adapting this example

To add more output routines without linking `printf` (which pulls in a lot
of library code):

```c
/* Print a newline */
putnl() { putstr("\r\n"); }

/* Print a hex byte */
puthex(n) unsigned n; {
    int d;
    d = (n >> 4) & 0xF;
    putchr(d < 10 ? '0'+d : 'A'+d-10);
    d = n & 0xF;
    putchr(d < 10 ? '0'+d : 'A'+d-10);
}
```

To use structs, arrays, or pointers: no restrictions — just remember to
declare all local variables at the top of each function before any
statements (K&R C requirement enforced by Micro-C).
